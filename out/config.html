Settings

You can configure your deploy script through configuration files Sanelotto_local.json on local machine and Sanelotto_server.json on server. Let's look at all properties at these files.

Sanelotto_local/Sanelotto_local.json

  • name — name of your project.
  • server — IP address of your server.
  • ssh_key — whether to use a authentication through SSH-key? true/false
  • login — your login on server.
  • pass — your password on server. When 'ssh_key: true' this property is irrelevance.
  • before_start_commands — do you need to execute commands from before.slfile before connection to server? true/false
  • before_start_dir — directory containing the before.slfile.
  • use_sudo — whether to use 'sudo' before all commands? true/false
  • logging — whether to save all logs to text files? true/false
  • server_dir — directory on server containing the Sanelotto_server.

Sanelotto_server/Sanelotto_server.json

  • name — name of your project.
  • github_user — username on GitHub.
  • github_repo — name of repository on GitHub.
  • branch — branch on your repository. 'Master' by default.
  • rewrite_configs — overwriting some files in project after downloading or updating through GitHub. true/false
  • configs_dir — directory containing the files for overwriting. Default: 'app'
  • first_commands — use system commands after first downloading from GitHub. true/false.
  • start_commands — use system commands after first downloading and updating from GitHub. true/false
  • stop_commands — use system commands after updating from GitHub. true/false
  • use_sudo — whether to use 'sudo' before all commands? true/false
  • logging — whether to save all logs to text files? true/false
  • commands_dir — directory containing .slfile files with commands. Default: 'files'

Keep in mind that configuration .json files must be located in Sanelotto_local or Sanelotto_server directories. You must run command sanelotto start [TYPE] from directory that contains Sanelotto_local or Sanelotto_server , not configuration file. Example:

Path on local machine

/home/username/apps/myproject/Sanelotto_local/Sanelotto_local.json

You must run sanelotto start local from myproject directory.

Property 'server_dir' in Sanelotto_local.json must contains absolutely path on server to directory, contained the Sanelotto_server . Example:

Path on server

/home/username/apps/myproject/Sanelotto_server/Sanelotto_server.json

In local machine file Sanelotto_local.json must contains line

"server_dir": "/home/username/apps/myproject/"
