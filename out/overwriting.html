Overwriting

Sometimes you need to change or create files in project before start without saving them on GitHub. These can be configuration files contains passwords or other private information. Sanelotto included a system for overwriting files in project to files in Sanelotto directory.

If overwrite mode is active, you can find 'files' directory in Sanelotto_server . There is the __main__.json inside it. This file contains information about files for overwriting. Information is recorder as follows:

"filename": "/absolutely/path/to/desired/place/newfile.extension"

The file is read from the Sanelotto_server/files/ and writes to path from __main__.json. Example:

__main__.json

{ "pass.json": "/home/username/apps/myproject/pass.json", "database_config.xml": "/home/username/apps/myproject/db/config/db_pass.xml", "Licence": "/home/username/apps/myproject/other_data/Full_license.txt" }

The directory Sanelotto_server/files/ must contained four files:

__main__.json pass.json database_config Licence

These files will be written to new addresses under new names.
